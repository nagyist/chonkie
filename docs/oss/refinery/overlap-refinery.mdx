The `OverlapRefinery` enhances chunks by incorporating context from neighboring chunks. This is useful for tasks where maintaining contextual continuity between chunks is important, such as question answering or summarization over long documents. It can add context as a prefix (from the preceding chunk) or a suffix (from the next chunk).

<Info>
  **Built into all chunkers:** Since `OverlapRefinery` is integrated into `BaseChunker`, all chunkers support overlap natively via the `chunk_overlap` parameter. You only need to use `OverlapRefinery` directly if you want to apply overlap as a separate post-processing step.
</Info>

## Overlap via Chunkers (Recommended)

The simplest way to add overlap is to pass `chunk_overlap` directly to any chunker:

```python
from chonkie import TokenChunker, SentenceChunker, RecursiveChunker

# Fixed token overlap
chunker = TokenChunker(chunk_size=512, chunk_overlap=50)

# Percentage-based overlap (10% of chunk size)
chunker = TokenChunker(chunk_size=1000, chunk_overlap=0.1)

# Works with any chunker
chunker = SentenceChunker(chunk_size=1024, chunk_overlap=128)
chunker = RecursiveChunker(chunk_size=2048, chunk_overlap=64)
```

All chunkers accept these additional overlap parameters via `**kwargs`:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `chunk_overlap` | `int \| float` | `0` | Overlap size. Int = absolute tokens; float (0-1) = fraction of chunk size. |
| `overlap_mode` | `"token" \| "recursive"` | `"token"` | How to calculate overlap boundaries. |
| `overlap_method` | `"suffix" \| "prefix"` | `"suffix"` | Where to add context: suffix appends from next chunk, prefix prepends from previous. |
| `overlap_rules` | `RecursiveRules` | `RecursiveRules()` | Rules for recursive overlap mode. |

## Standalone Usage

You can also use `OverlapRefinery` as a standalone post-processing step:

```python
from chonkie import TokenChunker, OverlapRefinery

text = "This is the first sentence. This is the second sentence, providing context. This is the third sentence, which needs context from the second."
chunker = TokenChunker(chunk_size=512)
chunks = chunker(text)

# Apply overlap as a separate refinement step
overlap_refinery = OverlapRefinery(
    chunk_overlap=50,
    overlap_method="suffix",
    overlap_mode="token",
)

refined_chunks = overlap_refinery(chunks)
```

### Recursive Mode

For more natural overlap boundaries, use recursive mode with custom rules:

```python
from chonkie import OverlapRefinery, RecursiveRules, RecursiveLevel

rules = RecursiveRules(
    levels=[
        RecursiveLevel(delimiters=["\n\n"], include_delim="prev"),
        RecursiveLevel(delimiters=["."], include_delim="prev"),
        RecursiveLevel(whitespace=True)
    ]
)

overlap_refinery = OverlapRefinery(
    chunk_overlap=0.25,
    overlap_mode="recursive",
    overlap_rules=rules,
    overlap_method="suffix",
)

refined_chunks = overlap_refinery(chunks)
```

## Parameters

<ParamField path="chunk_overlap" type="Union[int, float]" default="0">
  The size of the overlap context. If an `int`, it's the absolute number of
  tokens. If a `float` (between 0 and 1), it's the fraction of the maximum chunk
  token count.
</ParamField>

<ParamField path="overlap_mode" type='Literal["token", "recursive"]' default='"token"'>
  The mode for calculating overlap. `"token"` uses the tokenizer directly.
  `"recursive"` uses hierarchical splitting based on `overlap_rules`.
</ParamField>

<ParamField path="overlap_method" type='Literal["suffix", "prefix"]' default='"suffix"'>
  The method for adding context. `"suffix"` adds context from the *next* chunk
  to the end of the current chunk. `"prefix"` adds context from the *previous*
  chunk to the beginning of the current chunk.
</ParamField>

<ParamField path="overlap_rules" type="RecursiveRules" default="RecursiveRules()">
  The rules used for splitting text when `overlap_mode` is `"recursive"`. Defines
  delimiters and behavior at different hierarchical levels. See
  `chonkie.types.RecursiveRules`.
</ParamField>

<ParamField
  path="overlap_tokenizer"
  type="Union[str, TokenizerProtocol, None]"
  default="None"
>
  The tokenizer to use for calculating overlap size. Can be a
  string identifier (e.g., "character", "word", "gpt2") or a tokenizer instance.
  If `None`, falls back to the chunker's tokenizer (when used as a mixin) or
  character-level counting (when used standalone).
</ParamField>

## Deprecated Parameters

The following parameters still work for backward compatibility but are deprecated in favor of the new names:

| Deprecated | Replacement |
|------------|-------------|
| `context_size` | `chunk_overlap` |
| `mode` | `overlap_mode` |
| `method` | `overlap_method` |
| `rules` | `overlap_rules` |
| `tokenizer` | `overlap_tokenizer` |
| `merge` | Always merges context into `chunk.text` |
| `inplace` | Always modifies in place |
